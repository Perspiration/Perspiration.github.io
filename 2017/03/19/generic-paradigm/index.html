泛型总结

1. 什么是泛型

泛型本质上是将类型参数化，最常见的泛型的应用是List类，在使用List类时，我们会这么声明：List<String>，这里<String>就是泛型。

这里的E是一个类型参数，代表一种类型，它可以是Integer，String或者你定义的任何一种类型（但是不支持如int，double等基本类型）。在使用时，只需如此声明：List<String>。这里的String就是E的具体化。有了泛型之后，我们就不需要再编写类或者方法的时候指定类型了。一个泛型类可以支持多种类型，极大的减少了代码量。

泛型主要分两种，泛型方法和泛型类

5. 泛型的内部实现

5.1 类型转换

子类是可以替换父类的。当需要Object类的引用的时候，如果传入一个String对象是没有任何问题的（Object obj = “”;）。但是反过来的话，即用父类的引用替换子类引用的时候，就需要进行强制类型转换。
Integer[]是Number[]的子类型，但是ArrayList <Integer>不是ArrayList <Number>的子类型（ArrayList<Number> list = new ArrayList<Integer>();会报错），ArrayList<String>是List<String>的子类型（List<String> list = new ArrayList<String>(); 正确）

5.2 擦除

Java 语言中的泛型基本上完全在编译器中实现，由编译器执行类型检查和类型推断，然后生成普通的非泛型的字节码。这种实现技术称为 擦除（erasure）
擦除的过程：
第一步：找到用来替换类型参数的具体类。这个具体类一般是Object或者类型参数的上界
第二步：把代码中的类型参数都替换成具体的类。
第三步：去掉出现的类型声明，即去掉<>的内容。
比如T get()方法声明就变成了Object get()；List<String>就变成了List。

由于泛型类的擦除类型ArrayList<Integer>和ArrayList <String> 都将转换成ArrayList 类，因此ArrayList<Integer> 和ArrayList <String>实质上是同一个类。

6. 泛型的优点

一个最重要的好处是更强的类型检查，
另一个好处是消除了类型转换，这意味着可以用更少的代码，因为编译器确切知道集合中存储的是何种类型。
消除类型转换还可以节省时间，故使用泛型的性能和一般Java代码非常接近。
