内部类总结

内部类包含四种类型，分别是成员内部类、局部内部类、静态内部类和匿名内部类。

成员内部类

成员内部类定义在一个类的成员定义在类的内部。
基本形式如下：

class OutClass {
    private int a = 10;

    class InnerClass {
        public void print() {
            System.out.println("a = " + a);
        }
    }
}

成员内部类的使用依赖于外部类。也就是说，其他类要使用内部类，必须先初始化外部类。看下面的例子：

class Test {
    public static void main(String[] args) {
        OutClass out = new OutClass();
        OutClass.InnerClass inner = out.new InnerClass();
        inner.print();
    }
}

在上面的例子中有两个关键点：
1.使用InnerClass的格式是OutClass.InnerClass;
2.要初始化InnerClass，必须先初始化OutClass。

上面的例子中，我们使用InnerClass访问OutClass的成员变量，这是内部类的一种用法。看下面的例子：

class OutClass {
    private int a = 10;

    class InnerClass {
        private int a = 11;
        public void print() {
            int a = 12;
            System.out.println("a = " + a);
            System.out.println("this.a = " + this.a);
            System.out.println("OutClass.this.a = " + OutClass.this.a);
        }
    }
}

上面代码的输出是：

12
11
10

内部类可以访问外部类的成员变量以及方法，那这是如何实现的呢？
其实内部类在编译的时候，编译器会为内部类添加一个指向外部类对象的引用，这个引用在调用内部类构造方法的时候进行初始化。有了这个引用之后，内部类就可以通过它访问外部类的变量和方法了。这里也间接说明了为什么使用内部类时要先初始化外部类。

局部内部类

局部内部类定义在方法内部，也只能在所定义的方法内部进行访问。

静态内部类

静态内部类与成员内部类的区别在于使用static修饰类。静态内部类的实现不依赖于外部类，而且静态内部类只能访问外部类的静态变量。静态内部类的这些特性主要是因为它在编译时不会像成员内部类一样添加一个指向外部类对象的引用。具体使用看下面的例子：

class OutClass {
    private static int a = 10;

    static class InnerClass {
        public void print() {
            System.out.println("a = " + a);
        }
    }
}

class Test {
    public static void main(String[] args) {
        OutClass.InnerClass inner = new OutClass.InnerClass();
        inner.print();
    }
}

匿名内部类

匿名内部类，顾名思义这个内部类没有名字。既然没有名字，那么就很好理解它只能使用一次。下面是匿名内部类的一个例子：

abstract class Person {
    public abstract void eat();
}

public class Demo {
    public static void main(String[] args) {
        Person p = new Person() {
            public void eat() {
                System.out.println("eat something");
            }
        };
        p.eat();
    }
}

匿名内部类必须继承一个父类或者实现一个接口。有时候我们想要实现一个接口，但是这次实现不具有普遍性，只在这个地方这样实现。这种情况下我们就可以使用匿名内部类。

匿名类最主要的应用于多线程中实现Thread类或Runnable接口。如：

public class Demo {
    public static void main(String[] args) {
        Thread t = new Thread() {
            public void run() {
                for (int i = 1; i <= 5; i++) {
                    System.out.print(i + " ");
                }
            }
        };
        t.start();
    }
}

以及

public class Demo {
    public static void main(String[] args) {
        Runnable r = new Runnable() {
            public void run() {
                for (int i = 1; i <= 5; i++) {
                    System.out.print(i + " ");
                }
            }
        };
        Thread t = new Thread(r);
        t.start();
    }
}

还有一点要注意的是，当所在的方法的形参需要被内部类里面使用时，该形参必须为final。如下面的例子：

public class Test {
    public static void main(String[] args)  {

    }

    public void test(final int b) {
        final int a = 10;
        new Thread(){
            public void run() {
                System.out.println(a);
                System.out.println(b);
            };
        }.start();
    }
}

这里如果把变量a和b前面的任一个final去掉，这段代码都编译不过。

多重继承

内部类的作用：

  1. 访问外部类的成员变量
  2. 被外部类的方法操作
  3. 实现多重继承

其中最重要的应用就是实现多重继承，这也是内部类存在最主要的意义。内部类使得多重继承的解决方案变得完整。接口解决了部分问题，而内部类有效地实现了“多重继承”。

class Age {
    public int getAge() {
        return 20;
    }
}

class Name {
    public String getName() {
        return "Anna";
    }
}

class Person {
    public class MyName extends Name {
        public String getName() {
            return super.getName();
        }
    }

    public class MyAge extends Age {
        public int getAge() {
            return super.getAge();
        }
    }

    public int getAge() {
        return new MyAge().getAge();
    }

    public String getName() {
        return new MyName().getName();
    }
}

在上面的例子中，Person中定义了两个内部类MyAge和MyName，它们分别继承了类Age和Name，这样Person类也就拥有了Age和Name类的属性和方法。
